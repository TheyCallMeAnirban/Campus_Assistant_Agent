> **Type:** Synthetic Benchmark Dataset
> **Version:** 1.0
> **Generator Seed:** 42
> **License:** MIT (for benchmark/demonstration use)

## Purpose

This dataset was created to provide a **deterministic, statistically plausible benchmark corpus** for evaluating and demonstrating retrieval-augmented conversational AI systems.

It enables end-to-end testing of:
- Natural language intent classification (LangGraph)
- Deterministic Pandas-based quantitative filtering
- Section-level knowledge retrieval from structured markdown files
- Grounded LLM response generation (Gemini)

The corpus is **not** intended to represent official institutional data.

---

## Disclaimer

> All institution names in this corpus are entirely **fictional**. Any resemblance to real institutions is coincidental.
> No fees, placement statistics, rankings, or admission figures in this dataset represent any real institution.
> This dataset is intended solely for **demonstration, benchmarking, and evaluation** of AI systems.

---

## Generation Methodology

The corpus is generated by `knowledge/generate_dataset.py` using a deterministic rule engine with a fixed random seed (`SEED = 42`).

### Dependent Variable Cascade

All college facts derive from a single `tier_score` sampled from a type-specific distribution. This produces realistic correlations between attributes:

```
college_type
     │
     ▼
tier_score
     ├──▶ academic_score, campus_score, industry_score
     ├──▶ fees (interpolated from type range × tier)
     ├──▶ avg_package → highest_package (multiplier × tier)
     ├──▶ placement_rate (type range × tier)
     ├──▶ student_strength → hostel_capacity (ratio × strength)
     └──▶ campus_area (age_factor × type range)
```

No attribute is sampled independently. This ensures internal consistency.

### Rule Configuration

All distribution parameters live in `knowledge/generation_rules.json`. Modifying a distribution does not require touching Python code.

## Schema

### `college_info.csv`
| Column | Description |
|---|---|
| `college` | Fictional institution name |
| `city` | Real Indian city |
| `state` | Real Indian state |
| `type` | One of: iit_tier, nit_tier, iiit_tier, government, state_university, private |
| `fees_ug_inr` | Annual tuition fee (₹) |
| `placement_avg_lpa` | Average placement package (LPA) |
| `overall_score` | Aggregate benchmark score (0–10) |
| `academic_score` | Academic quality score (0–10) |
| `campus_score` | Campus infrastructure score (0–10) |
| `industry_score` | Industry engagement score (0–10) |

### Markdown Corpus (`corpus/{state}/{college_id}/`)
Each college has 8 section files:

| File | Content |
|---|---|
| `overview.md` | History, departments, location, affiliations |
| `fees.md` | Tuition, hostel, mess, payment details |
| `hostel.md` | Hostel capacity, facilities, mess details |
| `admissions.md` | Entrance exam, eligibility, process, documents |
| `placements.md` | Statistics, recruiters, internships |
| `scholarships.md` | Available schemes, application process |
| `campus.md` | Labs, library, sports, clubs, medical |
| `faq.md` | Common questions answered from structured facts |

## Validation Rules

Run `python knowledge/validate_dataset.py` to verify:

| Rule | Severity |
|---|---|
| `fee >= 0` | Hard error |
| `highest_package > avg_package` | Hard error |
| `hostel_capacity <= student_strength` | Hard error |
| `placement_pct` in 0–100 | Hard error |
| `overall_score` in 0–10 | Hard error |
| Fee outside expected type range | Warning |
| Missing markdown file | Warning |
| No placement row | Warning |

---

## Known Limitations

- Scholarship and placement data are statistically plausible but not representative of real outcomes.
- Hostel availability is set to "Yes" for all colleges. Real-world exceptions are not modelled.
- Admission cutoffs are not included in this version.
- Department seat intakes are not individually modelled.

---

## Reproducibility

```bash
python knowledge/generate_dataset.py   # generates corpus (seed=42)
python knowledge/validate_dataset.py   # validates all constraints
```

Running the generator twice with the same seed produces identical output. Changing `SEED` in `generate_dataset.py` produces a different but equally valid corpus.

## Intended Use

- Benchmarking retrieval-augmented generation (RAG) pipelines
- Evaluating LangGraph intent routing and node performance
- Demonstrating quantitative Pandas filtering ("colleges under ₹1.5 lakh")
- UI demonstration and frontend development
- System load testing and evaluation without real user data

## Out-of-Scope Use

- Publishing as factual information
- Any use that implies the data represents real institutions
